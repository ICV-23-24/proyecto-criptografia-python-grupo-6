Para la implementación del diseño en la pagina, hemos visto necesario utilizar 2 CSS, 1 general para toda la página con etiquetas básicas como el body y el footer, pero luego nos hemos visto precisados para un mejor diseño, uno específico para le cifrado Simetrico (el csimetrico.html) y otro para el login, donde va incluido tanto el inicio de sesión como el registro.
Gran parte del css del general va destinado para la documentación, ya que era lo que mas contenido contenía y donde más se podia intentar de atraer al usuario por la entrada visual.
Hay varios archivos con style independientes, se ha hecho de esta manera para poder individualizar más cada HTML sin necesidad tener muchos css distintos que puede causar confusión, aunque como he mencionado anteriormente, para la parte del cifrado simetrico, si se a visto necesario porque tiene una mayor cantidad de líneas de codigo, y de esta forma se tiene una mayor organización.
